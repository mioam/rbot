# 项目规范（轻量版）

## 1. 分支规范

只保留两个长期分支：

- `main`：部署/稳定分支
- `dev`：日常开发分支

1. 在 `dev` 开发，负责模型训练
2. 功能稳定后合并到 `main`，用于在真机部署
3. `main` 只接收来自 `dev` 的合并

## 2. Commit 规范

格式：

```text
type: 一句话说明
```

常用 type：

- `feat`：新功能
- `fix`：修复问题
- `refactor`：重构（不改功能）
- `docs`：文档
- `chore`：杂项
